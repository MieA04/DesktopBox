# DesktopBox 项目计划文档 V2

**版本**：v2.0  
**目标平台**：Windows 10  
**技术栈**：Tauri 2 + TypeScript + **原生前端三件套（HTML/CSS/JS）** + 轻量辅助库  
**项目代号**：DesktopBox

---

## 一、项目概述

DesktopBox 是一个桌面图标管理与系统监控辅助工具，采用独立应用窗口形式运行（隐藏原生桌面图标与任务栏），提供收纳盒分区、实时同步桌面图标、系统资源监控、内嵌 PowerShell 终端等功能。应用具有半透明亚克力/云母材质视觉效果，动画丝滑，低内存占用。

**核心设计哲学**：通过隐藏系统原生图标与任务栏，自建高性能前端界面，彻底规避与 `explorer.exe` 及 Wallpaper Engine 的冲突，实现极致稳定与流畅。

## 二、技术栈（最终确认）

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端核心 | **原生 TypeScript + HTML/CSS** | 无虚拟DOM，直接操作DOM与GPU加速属性，保证 <50ms 动画延迟与低内存占用 |
| 构建工具 | Vite | 极速开发体验，支持 TypeScript 与模块化 |
| 桌面框架 | Tauri 2 (Rust 后端) | 轻量、安全、高性能 |
| 窗口与透明 | Tauri 原生 + `window.setEffects` (Windows 11/10) / WinAPI fallback | 实现亚克力/云母效果；若系统不支持则使用 `backdrop-filter: blur` |
| 系统监控 | `sysinfo` crate (Rust) | 获取 CPU/GPU/内存占用，每秒推送至前端 |
| 进程列表 | `sysinfo` + `winapi` | 初期只读，提供进程名、PID、CPU/内存占用 |
| 文件系统监控 | `notify` crate (Rust) | 轮询桌面文件夹变化（200-500ms，可配置） |
| 终端集成 | `xterm.js` + Rust 伪终端（`portable-pty` 或 `cmd.exe` 桥接） | 支持基础指令与自定义指令集 |
| 持久化 | `serde` + `tauri-plugin-store` | 保存分区布局、用户设置、自定义指令集 |
| 轻量辅助库 | 可选：`lit-html` 或直接手写组件；状态管理用 `MobX` 或 `Valtio` | 提升开发效率但不牺牲性能 |

## 三、功能需求（分阶段实现）

### 🔹 初期版本（MVP）

| 模块 | 功能描述 |
|------|----------|
| **原生桌面图标隐藏与恢复** | 软件启动时隐藏系统桌面图标（通过 `FindWindow` + `ShowWindow(SW_HIDE)`）；关闭软件时自动恢复显示 |
| **原生任务栏隐藏与恢复** | 软件启动时隐藏系统任务栏（通过 `FindWindow` + `ShowWindow(SW_HIDE)`）；关闭软件时恢复 |
| **一键显隐主界面** | 全局快捷键（如 `Ctrl+Shift+D`）切换 DesktopBox 窗口的显示/隐藏 |
| **桌面图标实时同步** | 轮询 `%USERPROFILE%\Desktop` 文件夹（200-500ms，可配置），自动更新界面中的图标列表。支持文件/文件夹/快捷方式 `.lnk` 的图标提取与名称显示 |
| **收纳盒分区** | - 用户可创建、删除、拖拽调整分区（位置、大小）<br>- 分区内图标支持拖拽移动（跨分区）<br>- 分区大小改变时图标自动重新排列（流式布局）<br>- 分区布局自动保存（JSON 文件） |
| **系统资源监控（数字面板）** | 显示：当前时间、CPU 占用率（%）、GPU 占用率（%）、内存占用率（% 与已用/总量）。刷新频率：1 次/秒 |
| **进程列表（只读）** | 表格显示进程名、PID、CPU 占用率、内存占用。支持按列排序与即时搜索过滤 |
| **辅助命令行窗口（基础版）** | 内嵌终端（`xterm.js`），支持常用基础指令：<br>- `ls` / `dir`<br>- `cd`<br>- `mkdir`, `rm`, `echo`<br>- `tasklist`, `taskkill`（需谨慎实现）<br>- 允许用户配置自定义指令（保存到设置文件，如 JSON） |
| **性能基线** | 内存占用 < 500MB，动画触发延迟 < 50ms |

## 四、美术与交互设计

- **窗口样式**：无边框、可拖动、可调整大小；背景半透明亚克力（Acrylic）或云母（Mica）。优先使用 Tauri 2 的 `window.setEffects(['acrylic'])`，若不可用则 CSS `backdrop-filter: blur()` 降级。
- **动画**：分区展开/折叠、图标拖拽移动等需使用 CSS `transform` 与 `transition`（或 `Web Animations API`），确保触发响应 <50ms。所有动画需主动使用 `will-change` 与 `requestAnimationFrame` 优化。
- **主题**：初期支持浅色/深色跟随系统，后期可自定义主题色。
- **字体与图标**：使用系统默认无衬线字体（`Segoe UI`），图标采用纯 SVG 内联或轻量图标集（如 `lucide-static`）。

## 七、风险与应对

| 风险 | 应对措施 |
|------|----------|
| Tauri 2 + WebView2 透明窗口可能无法直接使用系统级亚克力效果 | 回退方案：CSS `backdrop-filter: blur(20px)` 配合半透明背景，模拟毛玻璃 |
| Rust PTY 在 Windows 上兼容性复杂 | 初期简化：直接创建隐藏的 `cmd.exe` 进程，通过管道读写（非全 PTY），支持基础非交互命令；后期替换为 `portable-pty` |
| 隐藏/恢复任务栏可能被某些安全软件拦截 | 提供手动恢复功能（如托盘菜单“重置任务栏”），并在文档中说明 |
| 图标拖拽时大量 DOM 操作导致动画掉帧 | 使用虚拟滚动与绝对定位 + CSS `transform` 移动占位符；拖拽结束时再重新排列 DOM 树 |
| 内存占用初期超标 | 采用内存分析工具（`perf`，`memory-profiler`）优化 Rust 侧数据传递（避免大量字符串克隆），前端使用 DOM 复用与对象池 |
| 前端原生开发导致代码可维护性降低 | 采用模块化设计（ES modules），使用 TypeScript 严格模式，状态管理采用轻量响应式库（如 `@preact/signals` 或 `valtio`） |
