## 一、项目概述

DesktopBox 是一个桌面图标管理与系统监控辅助工具，采用**嵌套式透明容器架构**：全屏透明无边框主窗口作为画板容器，多个独立亚克力材质模块（图标收纳盒、资源监控、命令行终端、进程列表等）自由悬浮于桌面之上。

**核心设计哲学**：采用全屏透明无边框主窗口作为画板容器，在桌面壁纸之上直接渲染亚克力材质模块，避免与系统桌面组件（explorer.exe）直接交互。桌面图标与任务栏的显隐由用户在 Windows 系统设置中手动管理。

本计划文档基于 [功能需求文档.md] 中定义的 **44 项功能需求** 及 **12 项非功能需求** 制定，所有需求优先级（P0/P1/P2/P3）已纳入里程碑规划。

## 二、技术栈（最终确认）

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端核心 | **原生 TypeScript + HTML/CSS** | 无虚拟DOM，直接操作DOM与GPU加速属性 |
| 构建工具 | Vite | 极速开发体验，支持模块化 |
| 桌面框架 | Tauri 2 (Rust 后端) | 轻量、安全、高性能 |
| 窗口架构 | 全屏透明无边框主窗口 + 独立模块容器 | 主窗口仅为画板，所有视觉由模块承载 |
| 窗口效果 | `window.setEffects` + CSS `backdrop-filter` 降级 | 模块亚克力/云母材质 |
| 系统监控 | `sysinfo` crate (Rust) | CPU/GPU/内存数据采集 |
| 进程列表 | `sysinfo` + `winapi` | 只读进程信息 |
| 文件监控 | `notify` crate (Rust) | 轮询桌面文件夹（200-500ms） |
| 终端集成 | `xterm.js` + Rust 子进程管道 | 基础指令支持 |
| 持久化 | `serde` + `tauri-plugin-store` | 模块布局、用户设置、自定义指令 |
| 辅助库 | 可选：`@preact/signals` 或 `valtio` | 轻量响应式状态管理 |

### 3.2 模块通用规范（对应 REQ-FRM-001~006）

每个模块（图标区、监控区、终端、进程列表）具备以下能力：

| 属性 | 实现方式 | 对应需求 |
|------|----------|----------|
| **拖拽移动** | 鼠标事件 + `transform: translate()`，拖拽结束更新位置并持久化 | REQ-FRM-001 |
| **缩放调整** | 右下角/边缘拖拽手柄，改变 `width`/`height`，最小尺寸限制 | REQ-FRM-002 |
| **混合模式（悬浮/贴靠）** | 拖拽释放时判断距屏幕边缘距离（阈值50px），触发贴靠；双击标题栏解锁 | REQ-FRM-003 |
| **层级管理** | 点击模块时提升 z-index，始终前置 | REQ-FRM-004 |
| **持久化** | 保存位置、尺寸、停靠状态、透明度/模糊值到 `store.json` | REQ-FRM-005 |
| **亚克力材质** | `backdrop-filter: blur(var(--blur-strength))` + `background-color: rgba()` | REQ-FRM-006 |
