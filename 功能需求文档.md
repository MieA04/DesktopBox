## 一、产品概述

DesktopBox 是一个桌面图标管理与系统监控辅助工具，采用全屏透明无边框主窗口作为画板容器，多个独立亚克力材质模块（图标收纳盒、资源监控、命令行终端、进程列表等）自由悬浮于桌面之上。桌面图标与任务栏的显隐由用户在 Windows 系统设置中手动管理，DesktopBox 不做自动干预。

**核心用户价值：**
- 摆脱杂乱的系统桌面图标管理方式，获得高度可定制的桌面工作区
- 实时监控系统资源状态
- 便捷的内嵌终端操作
- 与 Wallpaper Engine 等桌面美化工具完全兼容

## 二、术语表

| 术语 | 说明 |
|------|------|
| **模块 (Module)** | 桌面上的独立悬浮面板，包含特定功能（图标盒、监控、终端等） |
| **亚克力材质** | 使用 `backdrop-filter: blur()` + 半透明背景实现的磨砂玻璃视觉效果 |
| **悬浮模式** | 模块自由浮动在桌面上，可任意拖拽移动 |
| **贴靠模式** | 模块吸附到屏幕边缘（左/右/顶/底），自动调整尺寸 |
| **混合模式** | 悬浮与贴靠两种模式的统称，模块可在两者间切换 |
| **拖拽镜像** | 拖拽图标时跟随鼠标的半透明副本，而非移动原图标 |
| **图标分区** | 图标收纳盒内的网格区域，可划分为多个分区管理图标 |
| **基础指令** | 终端模块支持的基本 shell 命令（`dir`, `cd`, `mkdir` 等） |
| **自定义指令** | 用户通过 JSON 配置的快捷命令，可在终端模块中一键执行 |
| **开发调试覆盖层** | 开发环境下在背景层叠加状态信息面板，用于诊断窗口状态、快捷键触发等运行时信息 |

## 四、详细功能需求

### 4.1 系统集成功能

#### REQ-SYS-001：引导用户手动管理桌面图标（原自动隐藏）

| 字段 | 内容 |
|------|------|
| **ID** | REQ-SYS-001 |
| **标题** | 引导用户手动隐藏/恢复桌面图标 |
| **优先级** | P3（已从 P0 降级） |
| **描述** | 不再通过代码自动隐藏桌面图标。改为在用户手册或首次启动提示中告知用户通过 Windows 系统设置手动隐藏：桌面右键菜单 → 查看 → 显示桌面图标 |
| **实现方式** | 用户文档 + 可选的一次性启动提示 |
| **验收标准** | 1. 用户手册中明确说明桌面图标管理方式 2. 不再调用 Win32 API 操作桌面图标显隐 3. 软件启动/退出不干预桌面图标状态 |
| **降级原因** | Windows 10/11 桌面图标窗口层级因系统版本、Wallpaper Engine 等第三方软件而异，`FindWindow/ShowWindow` 方案在不同环境表现不一致，维护成本高 |

#### REQ-SYS-003：全局快捷键切换模块显隐

| 字段 | 内容 |
|------|------|
| **ID** | REQ-SYS-003 |
| **标题** | 全局快捷键切换子模块显隐 |
| **优先级** | P0 |
| **描述** | 通过全局快捷键（默认 `Ctrl+Shift+D`）一键显示/隐藏指定的子模块。窗口背景始终保持可见，仅切换模块层的显隐。模块隐藏后其位置和尺寸被持久化保留，恢复时还原 |
| **实现方式** | Tauri `global-shortcut` 插件注册热键 → Rust 后端向 WebView 发送 `app:toggle-modules` 事件 → 前端 `ModuleManager` 根据事件切换模块显隐（`display: none` / 恢复）。默认快捷键作用于所有模块，未来支持通过配置指定模块 ID 列表 |
| **验收标准** | 1. 快捷键全局生效（焦点在任意应用时均可触发） 2. 首次按下隐藏所有模块，再次按下恢复所有模块 3. 快捷键可在配置文件中自定义 4. 窗口背景始终透明可见，不随快捷键变化 5. 模块恢复时位置、尺寸与隐藏前一致 6. 模块隐藏后不响应鼠标事件 |
| **边界条件** | 所有模块已隐藏时再次按快捷键应全部恢复；模块恢复后保持原有层级顺序 |

### 4.2 模块框架功能

#### REQ-FRM-001：模块拖拽移动

| 字段 | 内容 |
|------|------|
| **ID** | REQ-FRM-001 |
| **标题** | 模块拖拽移动 |
| **优先级** | P0 |
| **描述** | 所有模块支持鼠标拖拽在桌面上自由移动 |
| **实现方式** | `PointerEvent` + `transform: translate()`，拖拽结束后持久化位置 |
| **验收标准** | 1. 鼠标按住模块标题栏可拖拽 2. 拖拽过程中模块跟随鼠标平滑移动 3. 松手后模块停留在释放位置 4. 重启软件后模块位置恢复 |
| **边界条件** | 模块不可拖出屏幕可见区域外；拖拽到边缘应触发贴靠判定 |

#### REQ-FRM-002：模块缩放调整

| 字段 | 内容 |
|------|------|
| **ID** | REQ-FRM-002 |
| **标题** | 模块缩放调整 |
| **优先级** | P0 |
| **描述** | 所有模块支持通过边缘/角落手柄拖拽缩放尺寸 |

#### REQ-FRM-003：模块悬浮/贴靠混合模式

| 字段 | 内容 |
|------|------|
| **ID** | REQ-FRM-003 |
| **标题** | 悬浮/贴靠混合模式 |
| **优先级** | P0 |
| **描述** | 模块支持自由悬浮和边缘贴靠两种模式，拖拽释放时自动判定 |
| **实现方式** | 释放时检测距屏幕边缘距离（阈值50px），触发对应贴靠行为 |
| **验收标准** | 1. 贴靠左边缘：模块吸附左侧，宽度 = 屏幕宽 × 0.25 2. 贴靠右边缘：模块吸附右侧 3. 贴靠顶部：模块吸附顶部，宽度 = 屏幕宽度 4. 贴靠底部：模块吸附底部 5. 不满足贴靠条件时保持悬浮 |
| **边界条件** | 贴靠后双击标题栏可解锁回到悬浮模式；贴靠模式下禁止拖拽 |

## 六、用例分析

### UC-001：日常桌面管理

```
角色：普通用户
前置条件：DesktopBox 已启动
主流程：
  1. 用户通过图标收纳盒模块浏览桌面文件
  2. 用户双击文件 → 系统默认程序打开
  3. 用户在桌面新建文件 → 收纳盒在 500ms 内自动显示新文件
  4. 用户拖拽图标在模块内重排顺序
后置条件：布局变化自动保存
```

### UC-002：系统资源监控

```
角色：开发者/性能敏感用户
前置条件：DesktopBox 已启动，资源监控模块可见
主流程：
  1. 用户扫视监控面板查看 CPU/内存使用率
  2. 发现内存占用过高
  3. 查看进程列表，按内存排序定位高占用进程
  4. 在终端中执行 taskkill 结束进程（或后期版本右键操作）
后置条件：无
```

### UC-003：快速终端操作

```
角色：开发者
前置条件：DesktopBox 已启动，终端模块可见
主流程：
  1. 用户点击终端激活
  2. 输入 `cd D:\projects` 切换目录
  3. 输入 `dir` 查看文件列表
  4. 输入自定义快捷指令一键执行常用操作
后置条件：当前工作目录保持
```

### UC-004：个性化桌面布局

```
角色：自定义爱好者
前置条件：DesktopBox 已启动
主流程：
  1. 用户拖拽图标收纳盒到桌面左侧，触发贴靠
  2. 缩放资源监控模块到合适尺寸，置于右上角
  3. 打开终端模块，置于右下角
  4. 调节各模块透明度和模糊到满意的视觉效果
  5. 关闭 DesktopBox 后再启动 → 布局完全恢复
后置条件：布局已持久化
```

### UC-005：快捷隐藏/恢复模块

```
角色：所有用户
前置条件：DesktopBox 正在运行，子模块可见
主流程：
  1. 用户按下 Ctrl+Shift+D
  2. 所有 DesktopBox 子模块隐藏，背景窗口保持透明可见
  3. 用户完成操作后再次按下 Ctrl+Shift+D
  4. 所有子模块恢复显示，位置和尺寸与隐藏前一致
后置条件：模块显隐状态持续到下次切换
```
